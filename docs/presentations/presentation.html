The problem
Every handoff loses something.
01
Specs arrive with assumptions baked in — and nobody asked the hard questions.
Vague success criteria. Missing edge cases. Scope that quietly balloons.
02
Design decisions get made silently in code.
Because nobody flagged the open question before engineering started.
03
QA finds in week 6 what a PM could have caught in week 1.
Context is lost at every phase boundary. The engineer never read the original why.
The idea
What if every phase had a world-class expert?
Phase 1 · PM agent
15 years shipping at world-class consumer and enterprise companies. Asks the uncomfortable questions. Won't let a vague spec through. Has seen every way a spec can fail.
Phase 2 · Design agent
Principal UX from a top-tier product company. Surfaces every interaction decision explicitly. No silent UI choices resolved in code.
Phase 3 · Architect agent
Distinguished Engineer from a hyperscale tech company. Knows which complexity is necessary and which is premature. Speaks plainly about tradeoffs.
Phase 5 · QA agent
Senior QA who thinks like an adversary. Every acceptance criterion gets a test. No happy-path-only coverage. No exceptions.

These aren't chatbots. They're opinionated specialists — and they read every upstream spec before acting.

Each phase is a Slack conversation between a human and an AI specialist. Each phase produces a spec file committed to GitHub. No phase starts until the previous one is explicitly approved.
Phase state lives entirely in GitHub — which branches exist, which spec files are merged. No external database. Any agent re-run on the same GitHub state produces the same result.
Key mechanism

Every draft runs through an audit gate

Result: OK
Draft saved

No conflicts, no gaps. Spec committed to the feature branch and presented for review.

Result: Conflict
Save blocked

Draft contradicts product vision or system architecture. Issue surfaced explicitly. Human must resolve upstream — the agent won't proceed until GitHub confirms it.

Result: Gap
Saved + flagged

Draft has an unresolved question. Saved as-is, but a required human decision is surfaced before the spec can be approved.

Powered by claude-haiku-4-5 — runs automatically after every agent response. This pattern is the standard all future agents inherit.

Design principles

What makes this different

01

Humans approve every gate

No phase advances automatically. AI drafts, surfaces, and asks — but the human always decides to advance. The bot is a collaborator, not a gatekeeper.

02

GitHub is the only source of truth

All specs, phase state, and history live in GitHub. No summary files, no cached copies. Agents read from the authoritative source on every call — never from memory.

03

Zero hardcoding — any team, any repo

Every product-specific coordinate lives in WorkspaceConfig, loaded from env vars. A new team onboards by changing .env only. No TypeScript changes required.
