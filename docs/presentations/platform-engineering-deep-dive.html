The problem
Every handoff loses something.
01
Specs arrive with assumptions baked in — nobody asked the hard questions.
02
Design decisions get made silently in code.
03
QA finds in week 6 what a PM could have caught in week 1.
Root cause: a process with no memory and no expert at every handoff point.
The idea
What if the best expert in the room
was always in the room?
→Not a chatbot. An opinionated specialist who has seen hundreds of specs fail.
→Reads every upstream decision before writing a single word.
→Won't advance until the hard questions have real answers.
The key differentiator: they push back. They won't accept vague answers. They've seen hundreds of specs fail. Unlike a consultant, always available and never politically biased.

Under the hood

Built to be fast, cheap, and hard to break

Efficiency — live today
Model tiering — Sonnet only where it earns it
Sonnetspec shaping · architectural reasoning
Haiku ×4classify · filter · audit · summarize overflow
Parallel GitHub reads — Promise.all()
All context (vision, architecture, design system, draft, cross-feature specs) loads simultaneously. Sequential loading would add 400–800ms per request for zero benefit.
Cross-feature loading — 10s timeout
Loading all other approved specs for coherence is an enhancement, not a hard dependency. Slow GitHub API? Agent proceeds without it.
Conversation overflow → Haiku summary + cache
When thread history exceeds the limit, older turns are summarized by Haiku — not discarded. Summary cached in-memory keyed by (threadTs, messageCount). History is append-only, so the cache entry never expires.
Robustness — live today
Graceful shutdown — drain before exit
request-tracker.ts counts in-flight requests. On SIGTERM, waits up to 6 minutes (exceeds Anthropic's 5-min API timeout) before exiting. Deploys never cut off a user mid-response.
Disk persistence — survives bot restarts
Conversation history and confirmed agents written to disk in dev. Bot restart mid-spec loses no state. GitHub spec draft compensates for any history loss.
Non-fatal preview generation
Design agent HTML preview failure never blocks a spec draft save. The spec is the critical output — the preview is an enhancement.
Known gaps → planned
Phase statere-read GitHub every msg → Redis cache
History storedisk → Redis (multi-instance)
Context limitsilent fail → warn at 70% capacity
The Haiku vs Sonnet split is the biggest cost lever. Haiku is ~20x cheaper per token. Any task that doesn't require multi-turn reasoning or architectural judgment uses Haiku. The parallel loading pattern means context load time is bounded by the slowest single GitHub read, not the sum of all reads. The conversation summarization cache is append-only so it never needs invalidation.

Key mechanism

Every draft runs through an audit gate

OK
Draft saved

No conflicts, no gaps. Committed to the feature branch.

Conflict
Save blocked

Contradicts product vision or architecture. Human must fix upstream — agent re-reads GitHub to verify.

Gap
Saved + flagged

Unresolved question. Saved as-is, required decision surfaced before approval.

Runs automatically after every agent response. The pattern all future agents inherit.

Conflict is a hard block — the agent won't proceed until GitHub confirms the upstream doc was actually updated. It doesn't take the human's word for it.

Design principles

What makes this different

01

Humans approve every gate

AI drafts, surfaces, and asks — the human always decides to advance.

02

GitHub is the only source of truth

All specs and phase state live in GitHub. Agents read from the source on every call — never from memory.

03

Zero hardcoding — any team, any repo

New team onboards by changing .env only. No code changes required.

These are enforced in code — CLAUDE.md won't let a change through that violates them.
The vision
An expert at every phase.
From first idea to shipped code.
Phases 1–3 live. 6 agents in build. The pattern is set.
The test suite is the hard gate before Phase 2 ships. Every behavior that exists today must be tested — that's done before the design agent starts. The pattern established in Phase 1 (conflict detection, audit gate, GitHub as truth) is the pattern every future agent inherits.
